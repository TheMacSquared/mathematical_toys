# Virtual Environment Setup

Projekt używa Python virtual environment do izolacji zależności.

## Szybki Start

```bash
# Aktywuj venv (rób to za każdym razem gdy pracujesz nad projektem)
source venv/bin/activate

# Teraz możesz uruchamiać Python komendy
python --version
pip list
```

## Pierwsze Uruchomienie (już zrobione)

```bash
# Stwórz venv (już zrobione)
python3 -m venv venv

# Aktywuj
source venv/bin/activate

# Zainstaluj zależności
pip install -r requirements.txt
```

## Dezaktywacja

Gdy kończysz pracę:
```bash
deactivate
```

## Dodawanie Nowych Pakietów

```bash
# Aktywuj venv
source venv/bin/activate

# Zainstaluj pakiet
pip install nazwa_pakietu

# Zaktualizuj requirements.txt
pip freeze > requirements.txt
```

## Uwagi

- `venv/` jest w `.gitignore` - nie commituj tego folderu
- Każdy developer musi stworzyć własny venv lokalnie
- Zawsze aktywuj venv przed pracą nad projektem
